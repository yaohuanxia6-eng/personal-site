3.1 区域 A：色卡选择

默认态
半透明白底 hover → 悬停态
透明度↑, Y-1px click → 选中态
高亮+阴影
  • 单选逻辑：同一时间只有一张色卡处于选中态
  • 基础色卡：8 张固定色卡，不可删除

Section 05

手机端与 PC 端交互差异

PC 端（>768px）

  • 色卡区：双列（基础色 + 收藏色并排）
  • 试管数：4 根全部显示
  • 操作栏：一行横排（步数左 + 按钮右）
  • 色卡内容：色块 + 中文名 + HEX 值
  • 悬停效果：色卡上移、试管上移
  • 占位卡：显示（收藏区空位可见）
  • 布局：色卡 25% + 试管 75%

手机端（≤768px）

  • 色卡区：单列（基础色均分高度，收藏色在上方）
  • 试管数：仅显示 2 根（隐藏试管 3、4）
  • 操作栏：两行居中（步数上 + 按钮下等宽铺满）
  • 色卡内容：色块 + 中文名 + HEX 值（溢出截断）
  • 悬停效果：无（触屏设备无 hover）
  • 占位卡：隐藏（CSS display:none）
  • 布局：色卡左 + 试管右，padding 60/20/40
  • 色卡区可滚动：超出时出现半透明白色滚动条
关键差异 手机端隐藏了试管 3 和 4（CSS display: none），但 JS 中 4 根试管的数据模型仍然存在。收藏色使用 order: -1 排到基础色上方。

6.1 动画设计原则

  • 一致性：所有 hover 过渡统一 0.2s ease，液面动画稍慢（0.3s）体现"液体感"
  • 克制性：不使用弹跳、旋转等花哨动画，保持简洁自然
  • 反馈性：每个可点击元素都有 hover/active 态变化，给用户操作确认感
  • 性能：仅动画 transform 和 opacity，避免触发 layout 重排（液面 height 是唯一例外，因为需要表达"填充"语义）
